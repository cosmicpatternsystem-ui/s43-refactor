# Human-AI Provenance

This project maintains a structured record of how the codebase evolves through human strategic direction and AI-assisted engineering.

## Principles
- Human controls secrets and sensitive approvals.
- AI assists architecture, review, audit, refactor planning, and documentation.
- Every material change should be traceable.
- Important releases should include hashes, compile/test results, and audit bundles.

## Provenance Records
Records are stored in `.provenance/` and include:
- architecture decisions
- patch chains
- verification outputs
- valuation notes
- release evidence
- handoff notes

## Commercial Relevance
A documented provenance trail increases trust, reviewability, maintainability, and resale value.
