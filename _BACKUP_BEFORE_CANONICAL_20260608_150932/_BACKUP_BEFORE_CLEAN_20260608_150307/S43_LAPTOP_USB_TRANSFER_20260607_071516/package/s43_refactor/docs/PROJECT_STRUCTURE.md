# S43 Project Structure

This document describes the expected structure of the S43 project.

## Core Files

`s43.py`

Main production source file.

## Release Backups

`release_backups/`

Contains immutable release copies and release manifests.

Expected examples:
```text
s43.py.final_YYYYMMDD_HHMMSS
S43_RELEASE_MANIFEST_YYYYMMDD_HHMMSS.txt

## Tools Archive

`tools_archive/`

Contains engineering utilities that were used for cleanup, audit, review, or extraction.

These tools are archived for reproducibility and should not be required for normal runtime.

## Packed Archive Backup

`archive_packed_backup/`

Contains compressed historical archives and their manifests.

Expected examples:

text
S43_ARCHIVE_PACK_YYYYMMDD_HHMMSS.tar.gz
S43_ARCHIVE_PACK_MANIFEST_YYYYMMDD_HHMMSS.txt
archive_original_YYYYMMDD_HHMMSS/

## Logs

`logs/`

Runtime logs generated by the application.

## Simulation

`simulation/`

Simulation scripts and experimental runtime checks.

## Tests

`tests/`

Unit tests, integration tests, and verification scripts.

## Docs

`docs/`

Project documentation, operational guides, and recovery instructions.
